# Smoke Checklist

Короткий список ручных проверок перед сдачей.

## 1. Запуск

- `npm install`
- `npm run dev`
- Открыть `http://localhost:5173`
- Убедиться, что заголовок и таблица видны

## 2. Загрузка счётчиков

- Первая страница загружается без ошибок
- В таблице есть строки (не `Нет данных для отображения`)
- Колонки заполнены: `№`, `Тип`, `Дата установки`, `Автоматический`, `Текущие показания`, `Адрес`, `Примечание`

## 3. Пагинация

- Нажать на страницу `2` (или другую)
- Убедиться, что `№` продолжается корректно по `offset`
- Вернуться на страницу `1`

## 4. Внутренний скролл

- Прокрутить таблицу вниз
- Проверить, что скролл у таблицы внутренний
- Проверить, что шапка таблицы остаётся на месте

## 5. Адреса

- В `Network` есть запросы `areas/?id__in=<id>`
- В колонке `Адрес` появляется полный адрес (`house.address` + квартира)
- Уже загруженные `areaId` повторно не запрашиваются на той же сессии

## 6. Удаление

- Навести курсор на строку
- Появляется кнопка удаления
- После удаления строка исчезает
- Таблица остаётся заполненной (если API возвращает достаточно данных)

## 7. Ошибки API

- При ошибке backend отображается блок `Ошибка: HTTP ...`
- Приложение не падает и остаётся интерактивным

## 8. Технические проверки

- `npm run lint`
- `npx tsc -b`
- `npm audit`

Ожидаемый результат: все команды проходят без ошибок и уязвимостей.
